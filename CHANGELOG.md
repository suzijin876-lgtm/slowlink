# 更新日志

本项目遵循语义化版本号。发布更新日志由本文件对应版本章节自动生成。

## [1.38.92] - 2026-07-17

- 修复管理员状态面板只因出现“定时注册”就触发注册公告规则的边界问题；现在优先读取独立的“注册状态”字段。
- `注册状态 | True/On/Enabled/1/开启` 仅放行到原正则继续判断，不强制转发；`注册状态 | False/Off/Disabled/0/关闭` 在原正则前拦截。
- `定时注册 | Turn off` 只表示定时功能关闭，不再当作当前注册关闭；因此“注册状态=True + 定时注册=Turn off”仍会按原规则转发。
- 保持无“注册状态”字段的自由/定时注册公告、注册码、抽奖、去重、监听和转发逻辑不变。

## [1.38.91] - 2026-07-16

- 修复同一抽奖先发布精简正文、随后补充“参与要求”并再次发布时因文本指纹不同而重复转发的问题；实际案例 `Lewa_movie/390202` 与 `Lewa_movie/390205` 现在会关联为同一场抽奖。
- 为“截止时间 + 奖品 + 发布群组 + 口令 + 活动详情”模板增加十分钟稳定关联身份，忽略容易后补或编辑的参与要求段。
- 标题、截止时间、奖品、发布群组、口令或活动详情任一稳定字段变化时仍视为新抽奖；超过十分钟后不再使用该短期关联身份拦截。
- 保持普通文本去重、抽奖 ID、随机种子、刮刮乐、注册码、监听、转发和重连逻辑不变。

## [1.38.90] - 2026-07-16

- 修正手动刷新群/频道列表时成功日志输出过早的问题：刷新结果通过完整性保护并写入缓存后才记录成功，不再出现“已刷新”后紧接“结果疑似不完整”的矛盾提示。
- 疑似不完整的刷新仍会保留旧列表并写入网页失败记录，但 Docker 日志只输出一条警告，不再为同一结果重复打印 `ERROR` 和 `WARNING`。
- 保持监听、识别、去重、转发、Telegram 高延迟重连和 CPU watchdog 行为不变。

## [1.38.89] - 2026-07-16

- 修正 V1.38.88 的 Whitelist 触发顺序：Whitelist 格式本身不再绕过主规则直接触发，必须先命中用户配置的正则或关键词。
- 主规则命中后继续提取固定十位 Whitelist 完整码，保留 `strong_whitelist` 身份用于完整码去重；相同码拦截，不同后缀放行。
- Whitelist 仍进入优先队列；九位、十一位、非 ASCII 字母数字、使用记录和关闭/无效上下文仍不会作为有效完整码转发。
- 网页说明同步改为“正则命中 -> 完整码提取 -> 去重 -> 转发”，不修改既有 Register/Renew 行为。

## [1.38.88] - 2026-07-16

- 新增固定十位纯字母数字的 `项目名[-更多项目段]-Whitelist_随机码` 内置强格式，例如 `WindMoon-Whitelist_1OTb0O0FMO`；无需主正则即可完整识别并直接触发转发。
- Whitelist 随机码保留原始大小写并生成独立完整码身份，相同码按“完整码重复”设置拦截，不同后缀互不影响。
- Whitelist 消息进入优先队列；九位、十一位、含下划线或缺少项目名前缀的近似文本不会作为完整码触发，使用记录和关闭/无效上下文继续由底层安全过滤拦截。
- 网页码识别区域补充 Whitelist 强格式说明；不修改现有 Register/Renew、正则、普通去重和转发逻辑。

## [1.38.87] - 2026-07-15

- 网页宿主机端口支持自定义：Docker Compose 使用 `SLOWLINK_WEB_PORT`，一键安装支持菜单输入和 `--port` 参数，端口会写入 `.env` 并在后续更新中保留；安装前会检查端口占用、显示占用详情并推荐空闲端口，不会停止现有服务。
- 修复一键安装在已有服务占用 `8080` 时构建后才失败的问题；基础依赖齐全时跳过 APT，Redis 通过健康检查后才构建应用，并将镜像构建与容器启动拆分为独立中文诊断步骤。
- 更新过程新增程序文件事务备份与失败回滚：复制新版、保存端口、构建、启动及完整验证任一步失败都会尝试恢复上一版；回滚本身失败时保留旧程序备份并显示路径，已有实例不能误走无备份安装；`.env`、Telegram Session、Redis 数据和用户配置不进入程序覆盖范围，首次安装失败只清理 `slowlink_app`。
- 安装完成前校验容器版本、Redis、网页 `/health` 和 CPU watchdog；`manage.sh status` 增加当前网页端口与访问地址，Docker Compose 等待 Redis 健康且仍只重建 `slowlink_app`。
- 运行诊断写入 Redis 改为尽力而为，Redis 短暂写入失败不再中断消息处理；消息 worker 增加状态统计、异常退出检测和自动补建，失败队列 worker 也纳入活跃任务计数。
- 轻量重连增加 worker 暂停门闩：等待当前正在处理的普通、优先及失败重试任务结束后再断开，新任务会留到重连完成后处理，避免转发发送途中被重连打断；移除两小时重连时多余的过期去重 key 扫描，不修改原有 FloodWait 重试语义。
- `/health` 同时检查 Redis、监听期望状态、监听线程、Telegram 连接、五分钟内心跳和 worker 完整性；期望运行但尚未真正就绪，或期望停止但监听仍存活时返回降级状态，便于 Docker 和安装器发现假健康。
- 缓存显示时区并支持保存后立即失效，未命中消息延后执行时间格式化；匹配器复用已规范化文本生成紧凑文本，减少高消息量下的重复处理，识别、正则、去重和转发业务规则保持不变。

## [1.38.86] - 2026-07-15

- 精简 Telegram 高延迟轻量重连日志：一次重连只记录准备重连、转发结果和重连结果，不再重复打印开始、完成或失败状态。
- 重连原因不再附带原始正则片段，仅保留延迟、来源和连续次数等必要诊断信息。
- 高延迟重连事件统一通过网页事件通道写入，网页日志与 Docker 日志使用同一条文案。
- 连续 30 分钟未收到消息时改用简短中文提示，并以最后一条消息或最后一次重连为计时基准，避免安静频道反复重连；高延迟阈值、缓存复用和转发业务逻辑保持不变。

## [1.38.83] - 2026-07-14

- 修复同一场跨群抽奖在“无随机种子”和“带随机种子”两种模板中分别走全文与稳定 ID 去重、导致重复转发的问题。
- 高置信普通抽奖会根据创建者、参与关键词、开奖时间和规范化奖品清单生成 10 分钟跨模板关联指纹。

## [1.38.79] - 2026-07-12

## [1.38.78] - 2026-07-12

- 多行遮罩注册码会为每一行生成独立文本去重指纹，不再因漏提取而错误合并。
- 自动迁移 Redis 中旧的 ASCII 纯码触发规则和码识别规则，无需手工修改正则。
- 未知中文后缀不会被截成半个邀请码参与去重，继续避免宽泛中文误识别。

## [1.38.75] - 2026-07-11

- 版本不一致时自动执行一次 `slowlink_app` 无缓存重建并再次校验，整个过程仍不停止 Redis 或其他服务。

- 手工填写且暂时无法解析的监听项自动退回原有过滤路径，避免优化导致漏消息；监听配置修改后无需重启即可刷新过滤集合。

- 应用启用 SIGUSR1 Python 全线程调用栈输出；watchdog 在首次高 CPU 和持续高 CPU 时保存调用栈、消息流和线程快照。
- watchdog 不再记录主机进程完整命令参数，避免诊断日志意外包含其他服务的 Token 或密码。
- 保留原有持续高 CPU 自动重启、Redis 数据、Session 和监听自动恢复行为；不修改匹配、去重或转发业务逻辑。
